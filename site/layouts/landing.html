Features

A single binary runs the whole CMS. Configure content types and fields in config.toml, extend with WebAssembly, and drive it from the terminal. See the full list →

One binary, no Docker

A static Go binary plus embedded SQLite. No runtime, no container, no node_modules in production.

Custom post types & fields

Define post types and fields in config.toml. The admin form, storage, and queries all read from it.

WebAssembly plugins

Drop a .wasm into plugins/ to hook before_save, after_create, after_publish.

Media & WebP

A media library with automatic WebP conversion, configurable thumbnail variants, and dedup.

AI built in

Image generation, text enhancement, and translation surfaced as buttons. Bring your own key.

i18n by default

Multilingual content with translation groups and hreflang in the sitemap.

SSR & one default theme

Server-rendered and SEO-friendly out of the box; customize via THEME_DIR.

REST API & CLI

A versioned /api/v1 with API keys, plus lesstruct-cli for agents and terminals.

Roles & security

Three roles, JWT auth, failed-login lockout, Argon2id, rate limits, and security headers.
